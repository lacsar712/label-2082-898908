# 校递快跑 (Campus Express Delivery) - Label 2082

一款专为在校大学生打造的轻量级、高颜值的全栈校园代取快递订单管理系统。
本项目基于 **C 语言构建原生高性能 RESTful 后端**，配合 **HTML5/CSS3/JavaScript 现代前端架构**，实现了从零依赖到完整容器化部署的工业级闭环。

---

## ✨ 核心特性

- 🔒 **完整的自建账户与安全体系**：
  - 基于 C 语言数组与二进制序列化实现的 Mock 数据库，支持脱机注册、登录校验。
  - 支持个人资料实时修改与密码安全更新（账户系统完全闭环）。
- 📦 **全生命周期的订单流转机制**：
  - 角色隔离双重视角：清晰区分「我发布的（雇主视角）」与「我接单的（跑腿视角）」。
  - 严谨的订单状态机流转：`待接单 (Pending)` ➔ `已接单/进行中 (Accepted)` ➔ `接单方确认送达 (Delivered)` ➔ `发布方确认验收/支付 (Completed)`。支持 `已撤回 (Cancelled)` 并可退市。
- 🔍 **智能检索引擎与前端动态渲染**：
  - 任务大厅集成「全网驿站匹配」，原生支持多维复合查询条件（支持 URL 级别中文解码）。
  - CSS3 玻璃拟态 (Glassmorphism) 与平滑微交互设计，响应式自适应卡片。
- ⚡ **无外部基建依赖 (Zero-Dependency)**：
  - 摒弃了沉重的 Node.js、MySQL 等中间件。所有数据（用户身份、订单及状态）直接通过 C 语言进行热内存处理及持久化同步至 `.bin` 二进制实体文件。
  - 包含自动守护兜底机制（容器重启不丢失业务数据）。

## 📂 项目结构

遵循标准的现代化全栈工程结构，前后端通过 RESTful 协议解耦分离：

```text
label-2082/
├── backend/
│   └── main.c           # 极简的高性能 C 语言 HTTP 源服务器代码
├── frontend/
│   ├── css/
│   │   └── style.css    # 独立 CSS 样式文件（定义全局和组件 CSS Variables）
│   ├── js/
│   │   └── script.js    # Vanilla JS (ES6) 前端路由、API 请求与核心交互处理
│   └── index.html       # 唯一 HTML 页面 (Single Page Application, 支持组件化切换)
├── data_orders.bin      # （运行时自动生成）订单关系图谱与详细载荷持久层文件
├── data_users.bin       # （运行时自动生成）用户证书及安全鉴权持久层文件
├── Dockerfile           # Alpine 极小镜像构建定义 (Static GCC compilation)
└── docker-compose.yml   # 业务调度编排与映射配置
```

## 🚀 启动指引

项目已被全面 Docker 化，且基于最轻量的 `alpine` 底层保障执行的隔离与一致性：

### 1. 本地执行编排构建
请确保您的环境已正确安装 [Docker](https://www.docker.com/)。
```bash

cd label-2082

# 2. 一键编译与拉起服务（后台常驻）
docker-compose up -d --build
```

### 2. 访问应用
构建完成后，直接打开浏览器访问系统主页面：
👉 [http://localhost:2082](http://localhost:2082)

## 🛠 技术深度解读

### 为什么选择原生 C + 裸 socket 作为 Web 服务器？
在不引入任何高层 Web Framework 的前提下，C 语言配合底层系统 API `sys/socket.h` 代表了计算极致的资源掌控：
- **静态资源分发**: 直接截获 HTTP 请求的头部 URI 向内穿梭读取 `frontend/`，模拟了轻量级 Nginx。
- **内存映射存储**: 利用 `fwrite` 与 `fread` 基于结构体的字节级偏移直接存取，绕过了关系型数据库的 SQL 解释层，在大并发弱存储设备上具备超群读取速度。
- **自定义 JSON Decoder**: 未内嵌第三方解析库，而是通过原生字符指针游历实现 JSON 实体摘取 (`strstr`, 偏移探针技术)。

### 前端 UI 的组件化管理
前端虽未使用 Vue/React，但通过原生 ES6 `let elements = {...}` 定义集中缓存了数十个 DOM 树节点，结合 `.hidden` 类名控制，完美模拟了现代 SPA 的路由导航机制与弹窗交互 (Toast & Modals) 回调，将重绘损耗降入极低阈值。

## 📄 API 通用设计文档

| 请求方法 | 端点 (Endpoint)        | 数据交互规范     | 功能描述                                   |
| :--- | :--- | :--- | :--- |
| `POST` | `/api/register`        | `JSON`   | 向网络分发注册新的全链路凭证与权限身份。   |
| `POST` | `/api/login`           | `JSON`   | 验证通信密码是否与加密通道一致，并颁布凭据。 |
| `GET`  | `/api/orders?[params]` | `QueryString` | 按标签 (category, creator, worker) 请求清单。|
| `POST` | `/api/orders`          | `JSON`   | 把需要跑腿的事件流分发进入中央调度总线。   |
| `POST` | `/api/update_status`   | `JSON`   | 执行生命周期内关键节点的状态更新流转动作。 |
| `POST` | `/api/update_profile`  | `JSON`   | 用户自决数据维护与鉴权证书的安全重置等。   |
